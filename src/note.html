vant: 下载：npm i vant -S 按需加载：npm i babel-plugin-import -D 引入：import { xxx } from 'vant'; 使用：Vue.use(xxx)； 注意的点：按需引入，提高网页性能，不用引入样式了 配置 babel-plugin-import 插件后，将不允许以全局方式导入组件，使用组件，引入css样式 分页： 分为两种： 分页的意义：数据太多， 1、后台处理数据，前端渲染 2、后台返回所有数据，前端处理（打后台） vue路由拦截：（路由导航钩子） 1：全局钩子： beforeEach、 afterEach     2：单个路由里面的钩子：  beforeEnter、 beforeLeave     3: 组件钩子：beforeRouteEnter、 beforeRouteUpdate、 beforeRouteLeave 作用：进行路由拦截，做权限控制，后台管理系统显现较多 路由导航钩子有三种： 1、全局导航 （常用） router.beforeEach():前置守卫，接收一个回调函数作为参数，此回调函数有三个参数 分别是to from next() to:去哪个导航 from:从哪里来 next()：回调(多次执行) router.afterEach():后置守卫，接收一个回调函数作为参数，此回调函数有两个参数 to:去哪里 from:从哪里离开 2、局部导航-路由独享守卫（常用） beforeEnter():局部组件路由守卫，进入组件之前调用进行拦截，卸载要去的那个组件路由配置中 beforeLeave:局部组件路由守卫，离开当前组件的路由守卫 3、组件内导航 （不常用） 路由拦截： 成功不拦截（直接调用next(),不传参），失败时拦截到你指定的页面去（next({path:'/xx'})）,去指定页面 路由元信息： 就是在路由中配置权限字段 vuex: vuex是存储公共数据的一个状态管理插件，可以实时依赖追踪 没有持久化(就是常驻内存，刷新页面不消失)，也就是刷新页面数据不丢失 vuex做持久化只存数据就可以 state this.$set()是组件内将静态数据转化为动态数据(Vue可追踪的数据) Vue.set():全局的将静态数据转换为响应式数据 面试题：组件内部， 打印台数据改变了，页面数据不改变，如何让页面数据改变，有哪些方法？ 回答：set(),this.$forceUpdate：强制更新视图(不能多次使用) 数据回显: 写游戏（用面向对象方式）
